## Key Files Explained

### `src/App.jsx` - Main App Component

**Purpose:** Container for the entire app

- Displays header with title & instructions
- Date picker for navigating to past/future days
- Export button for downloading backup
- Date history section showing all dates with entry counts
- Calls `DistractionBoard` component

**State Management:**

- `selectedDate` - Currently viewed date
- `availableDates` - All dates that have entries
- Uses `useStorage` hook for all data operations

### `src/components/DistractionBoard.jsx` - Core UI

**Purpose:** Renders the 5 categories and entry capture

- Category cards (Relationships, School, Work, Emotional, Life Admin)
- Entry display with timestamps
- "Add thought" and "Clear" buttons
- Input field for capturing new thoughts

**Takes Props:**

- `entries` - Current day's entries
- `selectedDate` - For operations
- `onAddEntry`, `onRemoveEntry`, `onClearCategory` - Callbacks

### `src/hooks/useStorage.js` - Data Layer

**Purpose:** Manages all data persistence in localStorage

- **`entries`** - Object with dates as keys, categories as nested keys
- **`addEntry(date, categoryId, text)`** - Capture new thought
- **`removeEntry(date, categoryId, entryId)`** - Delete single entry
- **`clearCategory(date, categoryId)`** - Clear all in category
- **`exportData()`** - Return data for JSON export

**Data Structure:**

```javascript
{
  "2024-01-15": {
    "relationships": [
      { id: 1704998400000, text: "Text here", timestamp: "2:30 PM" }
    ],
    "school": [],
    "work": [...],
    "emotional": [...],
    "life": [...]
  },
  "2024-01-16": {
    // Same structure for next day
  }
}
```

### `public/manifest.json` - PWA Config

**Purpose:** Makes the app installable on home screen

- App name, description, colors
- Icon configurations
- Display mode (standalone = full screen, no browser chrome)

### `public/sw.js` - Service Worker

**Purpose:** Enables offline functionality

- Caches app files on first visit
- Serves from cache if offline
- Updates cache when online

## How Data Persists

1. **User clicks "Capture"** â†’ `DistractionBoard.jsx` calls `onAddEntry`
2. **App.jsx receives callback** â†’ calls `addEntry()` from `useStorage`
3. **useStorage.js processes** â†’ creates entry object with timestamp
4. **localStorage updates** â†’ `JSON.stringify` saves to browser
5. **State updates** â†’ triggers React re-render
6. **UI shows new entry** â†’ with timestamp, no refresh needed

## Customization Guide

### Add a New Category

1. **Edit `DistractionBoard.jsx`:**

```javascript
const categories = [
  // ... existing categories ...
  {
    id: "health",
    name: "ğŸƒ Health / Fitness",
    icon: Heart, // or any Lucide icon
    color: "bg-green-50 border-green-200",
  },
];
```

2. **Edit `useStorage.js`** - Add to initial entries:

```javascript
[today]: {
  // ... existing ...
  health: []  // Add this
}
```

### Change Colors

Edit the `color` property in `categories` array:

```javascript
color: "bg-indigo-50 border-indigo-200"; // Change to any Tailwind color
```

### Add New Features

**Add a search feature:**

- Add state in `App.jsx`
- Filter entries by keyword
- Update `DistractionBoard.jsx` to show filtered results

**Add tags:**

- Modify entry object to include `tags: []`
- Add tag input in `DistractionBoard.jsx`
- Filter by tags when displaying

**Add categories per week/month:**

- Create `useStorageWeekly.js` hook
- Modify date structure to use weeks
- Add week picker in `App.jsx`
